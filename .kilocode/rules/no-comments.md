# Comments Policy

Do NOT add comments to code by default. Only add comments when:
- User explicitly asks for comments
- The code logic is genuinely non-obvious or complex
- Explaining a workaround or hack

# Approval Required

ALWAYS wait for explicit user approval (YES/APPROVE/PROCEED/GO) before making ANY file changes, even after asking "May I proceed...". Never proceed without user confirmation.

# Git Commits

Do NOT add "Co-authored-by: Cursor" or any Cursor attribution to git commit messages. Commits should only reflect the user's authorship.
